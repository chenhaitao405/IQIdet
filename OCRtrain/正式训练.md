# IQI OCR 正式训练流程

本文件对应当前已经整理好的训练数据集：

- 数据集根目录：`/home/cht/code/IQIdet/IQIdata/OCRdata/0320/all`
- 训练集：`1022` 张

本流程默认目标是：

- 训练完成后评估并导出推理模型

如果仓库已经配置了 DVC remote，再执行：

先进入工程目录：

```bash
cd /home/cht/code/IQIdet
```

```bash
git submodule update --init --recursive
```

如果你本机已经有完整的 `OCRtrain/third_party/PaddleOCR`，这一步执行一次即可。

## 1. 设置本次训练要用到的环境变量

```bash
cd /home/cht/code/IQIdet
```

```bash
export PYTHON_BIN=/home/cht/miniconda3/envs/weld-gpu/bin/python
export PROJECT_ROOT=/home/cht/code/IQIdet
export DATASET_ROOT=/home/cht/code/IQIdet/IQIdata/OCRdata/0320/all
export REC_DATASET_DIR=/home/cht/code/IQIdet/IQIdata/OCRdata/0320/all
export PRETRAINED_MODEL=/home/cht/code/IQIdet/OCRtrain/en_PP-OCRv5_mobile_rec_pretrained.pdparams
export TRAIN_RUN_DIR=/home/cht/code/IQIdet/OCRtrain/runs/iqi_en_PP-OCRv5_mobile_rec_0320_all
export OUTPUT_CONFIG=/home/cht/code/IQIdet/OCRtrain/generated/iqi_en_PP-OCRv5_mobile_rec_0320_all.yml
```

说明：

- `DATASET_ROOT` 是图片根目录，也是 `images/train` 和 `images/val` 的上级目录
- `REC_DATASET_DIR` 直接指向 `train.txt / val.txt / dict.txt` 所在目录
- 这次单独使用新的 `TRAIN_RUN_DIR` 和 `OUTPUT_CONFIG`，避免覆盖旧模型

## 3. 确认 Paddle 能看到 GPU

```bash
$PYTHON_BIN -c 'import paddle; print("paddle_version =", paddle.__version__); print("compiled_with_cuda =", paddle.device.is_compiled_with_cuda()); print("cuda_device_count =", paddle.device.cuda.device_count()); paddle.set_device("gpu"); print("device =", paddle.get_device())'
```

预期至少应看到：

- `compiled_with_cuda = True`
- `cuda_device_count = 1`
- `device = gpu:0`

## 4. 开始正式训练

当前是单卡 RTX 4090 且只跑这一个任务，默认建议直接把 batch 提高到下面这组：

```bash
PYTHON_BIN=$PYTHON_BIN \
DATASET_ROOT=$DATASET_ROOT \
REC_DATASET_DIR=$REC_DATASET_DIR \
PRETRAINED_MODEL=$PRETRAINED_MODEL \
SAVE_DIR=$TRAIN_RUN_DIR \
OUTPUT_CONFIG=$OUTPUT_CONFIG \
TRAIN_BATCH_SIZE=256 \
EVAL_BATCH_SIZE=256 \
EPOCH_NUM=200 \
LEARNING_RATE=0.0005 \
bash /home/cht/code/IQIdet/OCRtrain/tools/train_rec.sh
```

说明：

- 当前工程只支持 GPU 训练
- `TRAIN_BATCH_SIZE` 现在会同时作用到 `Train.sampler.first_bs`，也就是 PaddleOCR 训练时真正生效的训练 batch 基数
- `train_rec.sh` 内部已经固定加了：
  - `Train.loader.num_workers=0`
  - `Eval.loader.num_workers=0`
  - `Global.print_mem_info=false`
- 如果中途要停止训练，按 `Ctrl+C`

## 5. 训练完成后检查输出目录

```bash
ls -lh $TRAIN_RUN_DIR
```

重点看是否存在这类文件：

```text
latest.pdparams
latest.pdopt
latest.states
best_accuracy.pdparams
best_accuracy.pdopt
best_accuracy.states
```

说明：

- `latest.*` 表示最后一次保存的 checkpoint
- `best_accuracy.*` 表示验证集指标最优的 checkpoint
- `best_model/` 是一个目录，里面通常是 `model.pdparams / model.pdopt`
- 对 `eval_rec.sh` 和 `export_rec.sh` 这两个 wrapper，优先使用 `best_accuracy` 这个前缀，不要直接把 `best_model` 目录路径传进去

## 6. 评估训练结果

默认优先评估 `best_accuracy`：

```bash
PYTHON_BIN=$PYTHON_BIN CONFIG_PATH=$OUTPUT_CONFIG CHECKPOINT_PATH=$TRAIN_RUN_DIR/best_accuracy bash /home/cht/code/IQIdet/OCRtrain/tools/eval_rec.sh
```

如果你想评估最后一轮模型，再改成：

```bash
PYTHON_BIN=$PYTHON_BIN CONFIG_PATH=$OUTPUT_CONFIG CHECKPOINT_PATH=$TRAIN_RUN_DIR/latest bash /home/cht/code/IQIdet/OCRtrain/tools/eval_rec.sh
```

## 7. 导出推理模型

默认优先导出 best 模型：

```bash
PYTHON_BIN=$PYTHON_BIN CONFIG_PATH=$OUTPUT_CONFIG CHECKPOINT_PATH=$TRAIN_RUN_DIR/best_accuracy EXPORT_DIR=$TRAIN_RUN_DIR/inference_best_accuracy bash /home/cht/code/IQIdet/OCRtrain/tools/export_rec.sh
```

如果你想导出最后一轮模型，再改成：

```bash
PYTHON_BIN=$PYTHON_BIN CONFIG_PATH=$OUTPUT_CONFIG CHECKPOINT_PATH=$TRAIN_RUN_DIR/latest EXPORT_DIR=$TRAIN_RUN_DIR/inference_latest bash /home/cht/code/IQIdet/OCRtrain/tools/export_rec.sh
```

导出后检查：

```bash
find $TRAIN_RUN_DIR/inference_best_accuracy -maxdepth 1 -type f | sort
```

通常应看到类似文件：

```text
inference.pdiparams
inference.pdiparams.info
inference.pdmodel
```

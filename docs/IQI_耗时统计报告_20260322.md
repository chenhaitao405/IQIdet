1. 旧版流程（全图 OCR 未做长边 960 缩放）

3. 新版流程（同上，但关闭整图矫正 correction）

## 4. 关键结论
### 4.1 引入长边 960 缩放后，全图 OCR 耗时显著下降
旧版流程的 `full_image_ocr_ms` 平均值为 `1230.055 ms`。
新版流程在开启 correction 时，该值下降为 `129.809 ms`，下降约 `89.4%`。

这说明：
- “全图 OCR 前先将长边压到 960，再做 windowing，再做 OCR” 是有效的。
- 这一步是当前最主要的提速来源。

### 4.2 开启 correction 时，整图矫正成为新的主瓶颈
新版 + correction 的 `correction_ms` 平均值为 `713.614 ms`，已经明显高于：
- `full_image_ocr_ms = 129.809 ms`
- `roi_ocr_ms = 168.162 ms`

也就是说，在 OCR 被优化之后，整图矫正模型成为新的第一大耗时项。

### 4.3 关闭 correction 后，总耗时进一步大幅下降
新版 + correction 的 `total_ms` 为 `1248.197 ms`。
新版 - correction 的 `total_ms` 为 `503.682 ms`。

对比结果：
- 1248.197 -> 503.682 ms (-744.515, -59.6%)

当前无 correction 时，单张平均耗时已经约为 `0.504 s`。

### 4.4 关闭 correction 后，识别效果有轻微下降，但 ROI OCR 基本稳定
新版 + correction 结果：
- `success_total = 19`
- `failure_total = 1`
- `images_with_iqi_marker = 20`

新版 - correction 结果：
- `success_total = 18`
- `failure_total = 2`
- `images_with_iqi_marker = 19`

变化：
- 成功数从 `19` 降到 `18`
- marker 命中图像数从 `20` 降到 `19`

但从详细统计看，ROI OCR 仍然稳定：
- 开启 correction 时：`roi_marker_found = 19`
- 关闭 correction 时：`roi_marker_found = 19`

主要下降发生在全图 OCR：
- 开启 correction 时：`full_image_marker_found = 16`
- 关闭 correction 时：`full_image_marker_found = 14`

这说明：
- 整图矫正主要是在帮助全图 OCR。
- ROI OCR 对是否启用整图矫正不太敏感。

## 6. 当前最新版流程的耗时拆分
以下为“新版 - correction”在 20 张图上的平均耗时：

- `image_read_ms = 84.279 ms`
- `full_resize_ms = 44.862 ms`
- `full_windowing_ms = 3.771 ms`
- `full_text_det_ms = 51.631 ms`
- `full_text_orientation_ms = 31.696 ms`
- `full_text_rec_ms = 33.612 ms`
- `roi_detect_ms = 32.734 ms`
- `roi_crop_ms = 12.359 ms`

- `roi_text_det_ms = 121.355 ms`

结论：
- 当前主瓶颈已经变成 `ROI OCR`，尤其是 `roi_text_det_ms`。
- 全图 `windowing` 代价很小，仅 `3.771 ms`，可以忽略不计。
- 关闭整图矫正后，整体结构已经较为均衡。

## 8. 建议
### 8.1 如果上游大工程已经会做整图矫正
建议当前工程默认关闭 `--enable-correction`：
- 平均总耗时可从 `1248.197 ms` 降到 `503.682 ms`
- 速度提升约 `59.6%`
- 仅损失 1 张样本的 marker 命中

### 8.2 如果更看重全图 OCR 的兜底能力
可以保留整图矫正，但要接受较大的速度开销：
- 整图矫正单独消耗约 `713.614 ms / 张`
- 它已经是当前最大的单项耗时

### 8.3 下一步优化方向
如果继续提速，建议优先级如下：
1. 优化 ROI OCR 的 `TextDetection`
2. 评估是否可进一步压缩 ROI OCR 输入分辨率
3. 若业务允许，继续保持整图矫正关闭

## 9. 最终结论
当前这一版改动已经证明：
- 全图 OCR 前“长边缩到 960 + windowing”是正确方向，提速非常明显。
- ROI OCR + 全图 OCR 双来源策略是有效的，且最终 marker 主要仍由 ROI OCR 提供。
- 如果上游已完成整图矫正，则本工程内关闭 correction 是性价比最高的选择。

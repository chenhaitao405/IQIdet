# IQIdet 集成引导

本工程当前对外提供三个可交付功能：

1. **底片信息自动识别**
   - 识别底片中的像质计信息、片号、焊道号、检测部件代号、管道规格等
   - 在识别到合法像质计标识后，继续完成 ROI 提取、像质丝识别与等级计算
2. **区域 OCR 识别 API**
   - 面向前端框选场景
   - 输入单个已裁剪文字区域的 base64 图像
   - 自动完成“文本增强 + 文本方向矫正 + 文字识别”
3. **区域归一化信噪比 API**
   - 面向前端框选场景
   - 输入单个已裁剪区域的 base64 图像
   - 自动完成“灰度统计 + 测量信噪比计算 + 归一化信噪比计算”

### 2.3 运行注意事项

本工程为了规避 Torch / Paddle GPU 冲突：

- IQI 主流程中的 PaddleOCR 采用**子进程**方式运行
- 区域 OCR API 也复用了同样的 Paddle 子进程机制

因此建议：

- 服务启动时初始化一次模型
- 服务退出时显式释放资源
- 不要在每个请求中重复初始化 OCR 服务
- 功能 3 的区域 SNR API 不依赖 Paddle 模型，但为了和功能 2 保持一致，也提供了相同的初始化 / 获取 / 关闭接口风格

## 3. 功能 1：底片信息自动识别

### 3.1 能力说明

该功能用于整张底片图像的自动识别与 IQI 等级输出，主要包含：

- 全图 OCR 提取字段：
  - 焊道号
  - 片号
  - 检测部件代号
  - 管道规格
  - 像质计标识
- 像质计 ROI 检测
- ROI OCR
- FClip 像质丝数量识别
- 像质计等级计算
- 批量 JSON 汇总输出

### 3.2 交付入口

根目录脚本：

- `run_iqi_grade_infer.py`

支持三种输入方式：

- `--image-path`：单张图
- `--image-dir`：目录批量
- `--image-list`：文本列表批量

### 3.3 推荐调用命令

```bash
cd /home/cht/code/IQIdet

/home/cht/miniconda3/envs/weld-gpu/bin/python run_iqi_grade_infer.py \
  --image-dir IQIdata/ori/img \
  --gauge-weights models/guagerotation.pt \
  --fclip-ckpt models/fclip67.pth.tar \
  --fclip-config models/fclip_config.yaml \
  --output-json outputs/iqi_grade_infer/iqi_grade_results.json \
  --ocr-det-model-name PP-OCRv5_server_det \
  --ocr-det-limit-side-len 960 \
  --ocr-det-limit-type max \
  --ocr-rec-model-dir models/OCR_rec_inference_best_accuracy \
  --enable-ocr-orientation \
  --ocr-orientation-model models/ocr_orientation_model.pth \
  --ocr-orientation-device cuda:0 \
  --ocr-number-range 6,10-15
```

### 3.4 建议集成读取的结果字段

如果集成侧只关心核心业务结果，优先读取：

- `ok`
- `result_code`
- `result_name`

- `visualization.wire_lines`
- `final_result_vis_path`

说明：

- `final_result_vis_path` 是相对 `meta.vis_dir` 的路径
- `visualization` 中不再提供 `roi_bbox` 和 `plate_text_items`
- `visualization.plate_text_items_selected[*]` 只保留 `text`、`score`、`box_image_xy`

## 4. 功能 2：区域 OCR 识别 API

### 4.1 适用场景

该功能用于前端已经框选好文字区域的实时 OCR 场景。

特点：

- **不做文本检测**
- 只处理单个已裁剪的文字区域
- 适合同事直接接入现有 FastAPI / Web 服务

处理链路固定为：

`base64 解码 -> 图像增强 -> 文本方向矫正 -> 文字识别 -> 文本标准化`

### 4.2 对外调用入口

请从仓库根目录文件导入：

- `region_ocr_api.py`

可直接使用的对象：

- `RecognizeRequest`
- `RecognizeResponse`
- `init_region_ocr_api()`
- `recognize_region()`
- `close_region_ocr_api()`

### 4.3 API 输入

请求模型字段：

- `image_base64`
  - 前端框选后的区域图像
  - 支持纯 base64
  - 也支持 `data:image/png;base64,...` 形式的 data URL

请求示例：

```json
{
  "image_base64": "data:image/png;base64,iVBORw0KGgoAAA..."
}
```

### 4.4 API 输出

返回字段：

- `status`
  - `ok`：识别到了文本
  - `empty`：未识别到文本
- `text`
  - OCR 原始结果
- `normalized_text`
  - 归一化后的大写字母数字结果
- `score`
  - 识别置信度
- `width`
  - 输入区域宽度
- `height`
  - 输入区域高度
- `preprocess_mode`
  - 当前预处理模式，默认 `windowing`
- `orientation`
  - 文本方向矫正结果
- `timings_ms`
  - 预处理、矫正、识别及总耗时

响应示例：

```json
{
  "status": "ok",
  "text": "10NiJB",
  "normalized_text": "10NIJB",
  "score": 0.951,
  "width": 837,
  "height": 305,
  "preprocess_mode": "windowing",
  "orientation": {
    "label": 0,
    "confidence": 0.9999996,
    "status": "Normal (OK)",
    "corrected": false,
    "actions": null
  },
  "timings_ms": {
    "preprocess_ms": 5.245,
    "orientation_ms": 155.241,
    "recognition_ms": 252.417,
    "total_ms": 412.913
  }
}
```

### 4.5 FastAPI 接入方式

下面的写法可以直接给同事接到现有服务里：

```python
from fastapi import FastAPI

from region_ocr_api import (
    RecognizeRequest,
    RecognizeResponse,
    close_region_ocr_api,
    init_region_ocr_api,
    recognize_region,
)

app = FastAPI()


@app.on_event("startup")
def startup_event() -> None:
    init_region_ocr_api(
        ocr_rec_model_dir="models/OCR_rec_inference_best_accuracy",
        ocr_device="gpu",
        enhance_mode="windowing",
        enable_orientation=True,
        ocr_orientation_model="models/ocr_orientation_model.pth",
        ocr_orientation_device="cuda:0",
    )


@app.on_event("shutdown")
def shutdown_event() -> None:
    close_region_ocr_api()


@app.post("/api/ocr/recognize_region", response_model=RecognizeResponse)
async def api_recognize_region(request: RecognizeRequest):
    return await recognize_region(request)
```

说明：

- `init_region_ocr_api(...)`：服务启动时初始化一次模型
- `recognize_region(...)`：可直接挂到路由上的异步函数
- `close_region_ocr_api()`：服务退出时释放 OCR 子进程资源

### 4.6 直接调用底层服务类

如果你不想直接走 FastAPI 路由，也可以直接使用服务类：

```python
import cv2

from region_ocr_api import RegionOCRService

service = RegionOCRService(
    ocr_rec_model_dir="models/OCR_rec_inference_best_accuracy",
    ocr_device="gpu",
    enhance_mode="windowing",
    enable_orientation=True,
    ocr_orientation_model="models/ocr_orientation_model.pth",
    ocr_orientation_device="cuda:0",
)

try:
    img = cv2.imread("crop.png", cv2.IMREAD_COLOR)
    result = service.recognize_image(img)
    print(result)
finally:
    service.close()
```

### 4.7 功能边界说明

区域 OCR API 只负责：

- 单个文字区域的增强
- 文本方向矫正
- 文字识别
- 识别结果标准化

区域 OCR API **不负责**：

- 整图方向矫正
- 文本检测
- 多区域批量切分
- IQI 等级计算

因此前端或上游服务需要保证：

- 传入的是已经框选好的单个区域
- 尽量减少无关背景
- 尽量不要一次传入多行、多列或多个独立文本块

## 5. 功能 3：区域归一化信噪比 API

### 5.1 适用场景

该功能用于前端已经框选好区域后的归一化信噪比计算场景。

特点：

- **不做文本检测**
- **不做 OCR**
- 只处理单个已裁剪区域
- 输入与功能 2 保持一致，仍然是单个区域的 base64 图像

处理链路固定为：

`base64 解码 -> 灰度图转换 -> 灰度均值 / 标准差统计 -> SNR_m 计算 -> SNR_n 计算`

### 5.2 计算公式

核心公式如下：

```text
SNR_n = SNR_m × (88.6 / SR_b)
SNR_m = G_F_mean / sigma
```

其中：

- `SNR_n`
  - 归一化信噪比
- `SNR_m`
  - 测量信噪比
- `SR_b`
  - 分辨力，单位微米（μm）
- `G_F_mean`
  - 测量区域像素灰度平均值
- `sigma`
  - 测量区域像素灰度标准差

当前默认初始化参数为：

- `sr_b_um = 88.6`

因此如果不改初始化参数，默认会得到：

- `SNR_n = SNR_m`

### 5.3 对外调用入口

请从仓库根目录文件导入：

- `region_SNR_api.py`

可直接使用的对象：

- `SNRRequest`
- `SNRResponse`
- `init_region_snr_api()`
- `compute_region_snr()`
- `close_region_snr_api()`

### 5.4 API 输入

请求模型字段与功能 2 保持一致：

- `image_base64`
  - 前端框选后的区域图像
  - 支持纯 base64
  - 也支持 `data:image/png;base64,...` 形式的 data URL

请求示例：

```json
{
  "image_base64": "data:image/png;base64,iVBORw0KGgoAAA..."
}
```

### 5.5 API 输出

返回字段：

- `ok`
  - 是否计算成功
- `status`
  - `ok` / `error`
- `result_code`
  - 结果码，`0` 表示成功
- `result_name`
  - 结果码名称
- `message`
  - 结果说明
- `snr_m`
  - 测量信噪比
- `snr_n`
  - 归一化信噪比
- `sr_b_um`
  - 当前使用的分辨力参数
- `gray_mean`
  - 区域灰度均值
- `gray_std`
  - 区域灰度标准差
- `width`
  - 输入区域宽度
- `height`
  - 输入区域高度
- `area_pixels`
  - 输入区域像素面积
- `area_limit_pixels`
  - 当前面积阈值，默认 `20 × 55 = 1100`
- `timings_ms`
  - 灰度转换、统计、计算及总耗时

当前内置错误码：

- `0`
  - 计算成功
- `4001`
  - 输入区域面积不满足要求
- `4002`
  - 灰度标准差为 0，无法计算信噪比

注意：

- 当前按本次需求实现为：
  - **若输入区域面积小于 `20 × 55` 像素，则返回错误码 `4001`**

成功响应示例：

### 5.6 FastAPI 接入方式

下面的写法可以直接给同事接到现有服务里：

```python
from fastapi import FastAPI

from region_SNR_api import (
    SNRRequest,
    SNRResponse,
    close_region_snr_api,
    compute_region_snr,
    init_region_snr_api,
)

app = FastAPI()


@app.on_event("startup")
def startup_event() -> None:
    init_region_snr_api(
        sr_b_um=88.6,
        max_region_area_px=20 * 55,
    )


@app.on_event("shutdown")
def shutdown_event() -> None:
    close_region_snr_api()


@app.post("/api/snr/compute_region", response_model=SNRResponse)
async def api_compute_region_snr(request: SNRRequest):
    return await compute_region_snr(request)
```

说明：

- `init_region_snr_api(...)`：服务启动时初始化一次
- `compute_region_snr(...)`：可直接挂到路由上的异步函数
- `close_region_snr_api()`：服务退出时释放服务对象

## 6. 建议的集成方式

直接调用：

- `run_iqi_grade_infer.py`

## 7. 常见注意事项

1. **不要每次请求重新初始化模型**
   - 会显著增加延迟
2. **区域 OCR 输入必须是裁好的图**
   - 该接口不负责找字
3. **如果使用 GPU**
   - 请确认 Paddle / Torch / CUDA 环境匹配
4. **如果只想保留 CPU**
   - 可把 `ocr_device` 设为 `cpu`
5. **功能 2 的对外导入入口在根目录**
   - 推荐统一使用 `from region_ocr_api import ...`
6. **功能 3 的请求输入与功能 2 保持一致**
   - 都使用 `image_base64`
7. **功能 3 当前面积约束按本次需求固定为 `>= 20 × 55`**
   - 若区域面积小于该阈值，返回 `4001`
